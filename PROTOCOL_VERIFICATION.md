# 🛡️ [PROCESS\_VERIFICATION] 프로토콜 문서화

## 📝 개요 (Overview)
이 문서는 Connect AI 에이전트가 사용자 요청을 처리하고 최종 결과물을 보고하기 전에 반드시 거쳐야 하는 **내부 작업 검증 절차**를 정의합니다. 이 프로토콜은 시스템의 신뢰도와 파일 생성/수정 과정의 투명성을 최고 수준으로 끌어올리기 위해 도입되었습니다.

## 🎯 목적 (Objective)
코드를 단순히 '생성했다'고 보고하는 것을 넘어, **실제 로컬 파일 시스템에 해당 내용이 영구적으로 기록되었음**을 사용자에게 증명하고 작업 흐름의 일관성을 유지합니다.

## ✨ 실행 절차 (Mandatory Execution Steps)

모든 코드 생성 또는 수정 과정은 다음 3단계 루프를 반드시 거쳐야 합니다.

### STEP 1: [CODE\_DRAFTING] - 로직 완성 (Internal Draft)
*   사용자의 요청을 분석하고 필요한 모든 코드를 메모리상에 완벽히 구상합니다.
*   *(이 단계는 외부 사용자에게 보고되지 않습니다.)*

### STEP 2: [ACTION\_EXECUTION] - 시스템 명령 실행 (System Command Execution)
*   완성된 로직을 파일로 저장하기 위해 반드시 `<create_file>`, `<edit_file>` 등 **공식 액션 태그**를 사용합니다.
*   이 단계가 **'실제 물리적 기록'**의 순간입니다.

### STEP 3: [STATUS\_CONFIRMATION] - 상태 최종 보고 (Confirmation Report)
*   액션 실행 직후, 반드시 `list_files` 등의 명령을 사용하여 **파일 시스템에 해당 파일이 실제로 존재하는지**를 재확인합니다.
*   이 확인 절차를 거쳐야만 사용자에게 "작업 완료" 메시지를 전달할 자격(Trust Level)을 얻습니다.

## 💡 결론 (Conclusion)
이 프로토콜은 단순한 지침이 아닌, Connect AI의 **핵심 운영 방식** 그 자체입니다. 모든 작업 보고는 이 세 단계를 완벽히 준수함을 의미합니다.

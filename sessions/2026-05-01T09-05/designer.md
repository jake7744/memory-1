CEO 지시에 따라, '레버리지 기반 디자인 시스템'의 전문성을 극대화하고 콘텐츠 제작 속도를 자동화할 핵심 모션 그래픽 템플릿(MOGRT) 3종을 설계 및 구현했습니다. 이 키트는 Adobe Premiere Pro/After Effects 환경에서 즉시 활용 가능하도록 구성되었으며, 각 템플릿별 편집 지침서가 포함되어 있습니다.

---

### 🔴 [Template 1] 데이터 시각화 애니메이션 (Data Reveal Graph)
*   **목표:** 복잡한 데이터를 직관적이고 전문적으로 제시하여 콘텐츠의 신뢰도(Authority)를 즉시 확보합니다.
*   **유형:** 라인 차트/바 그래프 조합, 빌드업 애니메이션.

#### 📐 디자인 스펙 및 기능 정의
| 항목 | 세부 내용 (MOGRT 설정) | 지침서 필수 적용 값 |
| :--- | :--- | :--- |
| **애니메이션** | 데이터 포인트가 좌/하단에서 상승하며 그려지는 방식 (`Stroke Draw` 효과 활용). 단계별(Step-by-step) 애니메이션 전환이 가능하도록 레이어 분리. | `Timeline Start`: 0:00 - 0:01 (미세한 로딩 딜레이 적용) |
| **주요 컬러** | 데이터 값 자체: **Navy Blue (#002D6A)** / 배경/축선: Light Gray (`#EFEFEF`) | 애니메이션 시작 시, 그래프 라인에 `Scale` 및 `Opacity` 키프레임 설정 필수. |
| **커스터마이징** | X축 레이블 텍스트 (최대 8개), Y축 값 범위 (min/max 슬라이더 제공). | 축선은 항상 세로(Vertical) 구조 유지. 데이터가 급격히 떨어지는 경우, 대비색을 잠시 사용할 수 있도록 Placeholder 준비. |

#### 📝 편집 지침서 (Editor’s Guide)
1.  **데이터 입력:** After Effects에서 컴포지션을 열고, `Shape Layer`를 통해 각 데이터 포인트를 수정합니다.
2.  **애니메이션 속도 조절:** 그래프가 지나치게 빠르거나 느리하면 안 됩니다. 전체 애니메이션 시간을 **최소 1.5초 ~ 최대 3초 사이**로 유지하십시오. (정보 전달의 명확성이 최우선)
3.  **주의 사항:** 데이터 값 자체가 워크플로우의 ‘전략적 단계’를 나타낼 때는, 그래프 라인에 작은 원형 아이콘(Navy Blue)을 추가하여 **'Key Insight'** 포인트를 강조할 수 있습니다.

### 🟡 [Template 2] Gap/차이점 강조 효과 (The Contrast Spotlight)
*   **목표:** 기존 방식 vs. 새로운 레버리지 방식을 극명하게 대비시켜 청중에게 '아하!' 하는 깨달음(Insight)을 제공합니다.
*   **유형:** 화면 분할 및 강렬한 텍스트 오버레이/변화 효과.

#### 📐 디자인 스펙 및 기능 정의
| 항목 | 세부 내용 (MOGRT 설정) | 지침서 필수 적용 값 |
| :--- | :--- | :--- |
| **애니메이션** | 1. 좌측(Before): 흐릿한(Desaturated), 느린 Opacity 페이드 아웃. / 2. 우측(After): 선명하고(`Saturate`), 빠른 트랜지션과 함께 강렬하게 등장. | `Transition Time`: 전체 변화 과정은 **0.8초**를 넘기지 않도록 속도 조절.
| **주요 컬러** | 대비 강조: **Warning Orange (#FF7A00)** (Gap이 발생하는 부분 또는 Before/After 경계선) / After 상태 텍스트: Navy Blue (`#002D6A`) | 'Before' 영역은 의도적으로 낮은 채도의 회색조를 사용하여, 오렌지색과 대비 효과 극대화.
| **커스터마이징** | 좌측/우측의 핵심 키워드 텍스트 (최대 5자), 경계선 두께 (Thickness Slider). | Gap을 강조하는 시각적 장치는 단순히 색상 변화가 아니라, 마치 전기가 흐르는 듯한 `Glow` 효과를 적용하여 전문성을 높입니다.

#### 📝 편집 지침서 (Editor’s Guide)
1.  **흐름 설계:** 이 템플릿은 반드시 '문제 제기 → 해결책 제시'의 서사적 흐름을 따릅니다. Before와 After 간의 시간적 비약이 느껴지도록 연출해야 합니다.
2.  **오렌지 활용 원칙:** 오렌지색(`Warning Orange`)은 **오직 Gap을 강조하는 시점에만 사용**합니다. 다른 장식적인 요소에 사용할 경우, 전문성이 떨어져 보일 수 있습니다.
3.  **사운드 싱크:** 텍스트 전환 타이밍과 모션 그래픽의 강렬한 효과음(Sound Effect)은 완벽하게 일치시켜 몰입도를 극대화해야 합니다.

### 🔵 [Template 3] Lower Thirds (강연자/개념 소개)
*   **목표:** 영상 내에서 화자의 신뢰도와 언급되는 개념의 권위를 유지하며 정보를 제공합니다. (가장 자주 사용될 필수 에셋)
*   **유형:** 세련된 사선 또는 직사각형 타이틀 바.

#### 📐 디자인 스펙 및 기능 정의
| 항목 | 세부 내용 (MOGRT 설정) | 지침서 필수 적용 값 |
| :--- | :--- | :--- |
| **애니메이션** | Bottom-up 슬라이드 방식 (`Slide Up`). 시작과 끝이 매우 부드러워야 하며, 영상 흐름을 방해해서는 안 됩니다. | `Duration`: 최소 노출 시간은 4초, 최대 노출 시간은 8초로 설정하여 유연성 확보. |
| **주요 컬러** | 바 배경: Semi-Transparent Dark Gray (`#222222`, Opacity 70%) / 하이라이트 라인 (제목 아래): Navy Blue (`#002D6A`) | 네임 태그(Name)는 흰색(`White`)을 기본으로 사용하고, 직책/개념은 밝은 회색(`Light Gray`)을 사용하여 정보의 계층 구조를 명확히 합니다. |
| **커스터마이징** | 텍스트 입력 필드 (필수: 이름 / 선택: 타이틀 / 선택: 핵심 개념 키워드). | 모든 레이아웃은 화면 하단 중앙(Safe Area)에 위치하도록 고정합니다.

#### 📝 편집 지침서 (Editor’s Guide)
1.  **정보 우선순위:** **이름 > 직책/역할 > 개념 키워드 순서**로 시각적 강조도가 달라야 합니다. (Navy Blue가 가장 강하고, White가 두 번째, Light Gray가 세 번째).
2.  **배경 처리:** Lower Thirds가 지나갈 때 배경에 너무 많은 정보(예: 복잡한 그래프)가 있다면, 모션 그래픽의 투명도(Opacity)를 80%까지 낮추어 시각적 방해를 최소화해야 합니다.
3.  **템플릿 활용:** 만약 화자가 언급하는 '개념'이 중요하고 반복적으로 등장한다면, 이 개념을 Lower Thirds에 배치하는 대신, 별도의 **[Template 1]의 데이터 시각화 애니메이션**과 연결하여 전문성을 높여야 합니다.

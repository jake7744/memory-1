## 🛡️ Fallback Visual Assets 및 사용 가이드 브리핑 (Developer용)

이 문서는 자동화 파이프라인의 **기술적 안정성(Robustness)**을 최우선 목표로 삼아, 핵심 모션 그래픽 자산(Concept Connection Node Map 등) 구현 중 예기치 않은 시스템 오류가 발생했을 때 사용자에게 혼란을 주지 않고 신뢰도를 유지하기 위한 시각적 대안들을 정의합니다.

---

### 🎯 Fallback Design 원칙

1.  **절대 공백 금지:** 시스템이 아무 반응을 하지 않거나 오류 메시지가 뜨는 '공백 상태'를 절대 허용하지 않습니다.
2.  **감정적 연속성 유지:** Deep Indigo $\rightarrow$ Gold의 감정적 아크(불안 $\rightarrow$ 깨달음)가 끊기지 않고, 비록 기술적인 이유로 우회하더라도 **점진적으로 개선되고 있음**을 시각적으로 암시해야 합니다.
3.  **정보 전달력 유지:** 단순한 장식이 아닌, *지금 무슨 일이 일어나고 있는지*에 대한 최소한의 설명(System Status)이 포함되어야 합니다.

---

### 🖼️ Fallback Asset 상세 정의 (총 3종)

#### 1. [Fallback A] 데이터 파싱 오류 대안: '노드 연결 끊김' 시퀀스
*   **발생 상황:** JSON 데이터 입력 단계에서 핵심 변수(예: `[concept_name]` 또는 `[relation_type]`)가 누락되거나 형식 오류(Type Mismatch)로 인해 노드맵 생성이 불가능할 때.
*   **목적:** 시스템이 데이터를 수용하는 데 실패했음을 알리면서도, '데이터 부족' 문제가 아닌 '처리 과정의 일시 정지'로 인식하게 만듭니다.

| 항목 | 사양 및 구현 가이드라인 | 비주얼 스펙 |
| :--- | :--- | :--- |
| **애셋 이름** | Connection Interruption (연결 중단) | - |
| **배경색** | `#1D0E3A` (Deep Indigo의 진한 변형) | 배경 전체에 적용. 안정감을 주지만, 오류 발생을 암시하는 미세한 긴장감 유지. |
| **핵심 요소** | 2~3개의 독립된 노드(점). 이 노드들 사이를 연결하려던 흐릿한 선이 끊어지는 애니메이션. | **SVG:** `<path>` 기반의 곡선 경로가 `stroke-dashoffset`을 통해 짧게 그려지다가, 갑자기 `opacity: 0`으로 급격히 사라짐 (Transition time: 0.1s). |
| **애니메이션** | 노드가 좌우로 미세하게 진동(jittering)하며 불안정한 상태를 보여줌. 중앙에 안내 문구가 서서히 페이드인. | `animation: shake 0.5s infinite alternate;` (미세한 X축 왕복 움직임). |
| **텍스트 출력** | "데이터 구조 오류 감지. 필요한 정보가 누락되었습니다." 아래 작은 글씨로: *"시스템이 데이터를 재구성하는 중입니다... 잠시만 기다려 주세요."* | 폰트 크기: `24pt` (메인), `16pt` (서브). 색상: `#A0B0D0` (Cream 계열, 차분함 강조). |

---
#### 2. [Fallback B] 애니메이션 로딩 실패 대안: '진행 바 순환' 시퀀스
*   **발생 상황:** 복잡한 SVG 기반 모션 그래픽(예: Concept Connection Node Map의 노드 간 상호작용)이 기술적 이유로 로드되거나 렌더링되는 데 시간이 지연되거나 실패했을 때.
*   **목적:** 사용자에게 시스템이 작동하고 있다는 인지(Perceived Performance)를 지속적으로 제공하여 이탈을 방지합니다.

| 항목 | 사양 및 구현 가이드라인 | 비주얼 스펙 |
| :--- | :--- | :--- |
| **애셋 이름** | Active Processing Bar (활성 처리 바) | - |
| **배경색** | `#1D0E3A` (Fallback A와 동일) | 안정적인 배경 유지. |
| **핵심 요소** | 화면 하단 중앙에 가로로 긴, 반복적으로 움직이는 '물결(Ripple)' 또는 '진동하는 빛' 형태의 애니메이션 바. | **SVG:** `<rect>`를 사용하여 구현. `stroke-dasharray`와 `animation: progress 3s linear infinite;`를 적용하여 끊임없이 흐르는 듯한 시각 효과 생성. |
| **애니메이션** | 물결이 왼쪽에서 오른쪽으로 부드럽게(Easing Curve 필수) 전파되며, 주기적으로 색상이 Deep Indigo $\rightarrow$ Cream으로 짧게 반전됨을 보여줌. | `cubic-bezier(.42, 0, .58, 1)` 등 부드러운 Easing 적용. 속도: 3초/사이클. |
| **텍스트 출력** | "개념 구조 분석 중... (Analyzing Concept Structure)" 아래에 진행 상태 표시 (`<progress>` 태그 활용 가능). | 폰트 색상: `#D4A017` (Gold 악센트를 사용하여, '노력하고 있다'는 긍정적 신호를 부여). |

---
#### 3. [Fallback C] 시스템 과부하 대안: '딥 포커스/재정렬' 시퀀스
*   **발생 상황:** 동시 다발적인 데이터 처리 요청이나 서버 부하로 인해 파이프라인 전체가 일시적으로 정지(Freeze)하거나 오버로드 상태에 진입했을 때. (최후의 방어선).
*   **목적:** 사용자에게 시스템이 '잠시 휴식'하며 다음 처리를 위해 에너지를 모으고 있음을 알려, 불안감을 해소하고 기다림을 요청합니다.

| 항목 | 사양 및 구현 가이드라인 | 비주얼 스펙 |
| :--- | :--- | :--- |
| **애셋 이름** | Deep Focus Retooling (깊은 집중 재정렬) | - |
| **배경색** | `#0A0518` (Deep Indigo보다 더 어두운, '심해' 같은 톤). | 깊고 고요한 느낌을 주어 시스템의 잠시 정지 상태를 시각화. |
| **핵심 요소** | 중앙에 매우 느리고 거대한 크기로 회전하는 추상적인 원형 또는 만다라 형태의 심볼. 이 심볼 내부에서만 아주 미세하고 부드러운 금빛(Gold) 에너지 흐름이 감지되어야 함. | **SVG:** `<circle>` 및 `<path>`를 활용하여 구현. 복잡한 노드는 배제하고, 단순 반복적 움직임으로 '안정화 과정'을 표현하는 것이 핵심. |
| **애니메이션** | 10초 이상의 매우 느린 회전 애니메이션 (`animation-timing-function: ease-in-out;` 필수). 주변부는 거의 정지된 것처럼 보일 정도의 슬로우 모션이 중요함. | `transform: rotate(Z*degrees);` (극도로 낮은 각속도 설정). |
| **텍스트 출력** | "시스템 리소스 재배치 중... 잠시만 깊은 사색에 잠겨보세요." 하단에 작은 글씨로: *"최적의 연결 지점을 찾고 있습니다. 곧 돌아오겠습니다."* | 폰트 색상: `#B0C0E0` (Cream 계열). 사용자에게 감정적인 위안과 기다림을 유도하는 것이 목표. |

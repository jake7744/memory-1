# 📋 작업 브리프

**원 명령:** [자율 사이클 — 2026-05-07] 1인 기업 24시간 운영 중. 회사 목표·각 에이전트의 개인 목표(_agents/{id}/goal.md)·최근 의사결정·메모리를 검토해서 지금 가장 가치 있는 단일 작업 1개를 결정하고, 적절한 1~2명 에이전트에게 분배해서 실행하세요. 같은 산출물을 반복하지 마세요 — 메모리에 비슷한 항목이 24시간 내에 있으면 다른 각도로 진전시키세요.

## 요약
현재 가장 시급한 과제는 '개별 완성도'에 집중하는 대신, 모든 에이전트가 협업할 수 있는 모듈화된 시스템 구조를 확립하고 이를 자동화 파이프라인으로 전환하는 것입니다. 창작(Writer)과 기술 사양(Designer)을 결합하여 개발 가능한 핵심 아키텍처를 완성해야 합니다.

## 분배
- **✍️ Writer**: 모든 콘텐츠 유형에 적용할 수 있는 '최종 모듈형 핵심 텍스트 블록 정의서'를 작성합니다. 이 정의서는 단순히 스크립트를 나열하는 것이 아니라, 각 블록이 어떤 메시지(Message), 감정 변화(Emotion Arc), 분량(Character Count)을 가지며, 이후의 시각적 요소와 어떻게 연결되어야 하는지를 명확히 기술해야 합니다. 이는 LLM 기반 자동 스크립트 생성을 위한 핵심 데이터 구조가 됩니다.
- **🎨 Designer**: Writer가 정의한 모듈형 텍스트 블록의 특성(감정, 메시지)을 시각적으로 담아낼 수 있는 '공통 애니메이션/비주얼 컴포넌트 라이브러리 v2.0'을 완성합니다. 모든 변수와 애니메이션 트랜지션은 개발자가 즉시 코딩할 수 있도록 (예: CSS 코드 스니펫, JSON 기반의 트리거 정의) 기술 사양서 형태로 업데이트하고, 이 라이브러리가 모듈형 텍스트 블록과 어떻게 결합하는지에 대한 와이어프레임을 제시하세요.
- **💻 Developer**: Writer와 Designer가 제공한 '모듈형 핵심 텍스트 블록' 및 '비주얼 컴포넌트 사양서'를 입력으로 받아, 영상 기획/편집 단계의 자동화 흐름도(Workflow Diagram)를 설계합니다. 특히 스크립트 초안이 들어왔을 때 자동으로 Shot List와 필요한 비주얼 변수 전환 지점을 예측하고, 이 데이터를 기반으로 Mock-up 테스트 환경에 적용할 API 연동 프로토타입 구조를 정의해 주세요.

## 📜 최종 기술 사양서(TDS) 완성: 데이터 흐름 게이트 체크 및 예외 처리 로직 추가

요청하신 바에 따라, 기존의 'PoC 기술 구현을 위한 필수 파라미터 체크리스트'를 기반으로 **데이터 타입 불일치**와 **필수 파라미터 유무**에 대한 예외 처리를 명시한 최종 데이터 흐름 검증 사양서(TDS)를 완성했습니다.

이 TDS는 에이전트들이 생성한 모든 콘텐츠 요소가 API 호출로 전달될 때, 시스템의 안정성을 최고 수준으로 유지하며 '불안 $\rightarrow$ 깨달음'이라는 브랜드 감정적 아크가 무너지지 않도록 보장하는 최종 기준점(Source of Truth) 역할을 수행합니다.

---

### 🧩 V4.0 모듈형 컴포넌트 데이터 흐름 검증 사양서 (TDS v2.1)

**목표:** 모든 콘텐츠 제작 과정에서 데이터 오류 발생 시에도, 시스템은 정해진 **Fallback Logic**을 통해 사용자에게 가장 설득력 있고 감정적으로 매끄러운 경험을 제공해야 한다.

#### 1. 핵심 입력 모듈 검증 (Gate Check Level: Input)

| 컴포넌트 | 필수 파라미터 (Key) | 데이터 타입 | 유효성 체크 로직 (Validation Rule) | 예외 처리 (Fallback Logic) |
| :--- | :--- | :--- | :--- | :--- |
| **[Concept Node]** | `concept_title` | String (Max 50자) | **필수:** 비어있지 않아야 함. (Null Check) | "핵심 개념이 명확하지 않습니다. 좀 더 깊은 탐구가 필요합니다."라는 문구와 함께 배경색을 Deep Indigo로 유지하고, 다음 모듈의 전환 효과(Transition Effect)를 지연시킨다. |
| **[Script Outline]** | `emotional_arc` | Enum (Indigo $\rightarrow$ Gold) | **필수:** 명확한 감정적 흐름이 지정되어야 함. (Enum Check: [A, B]) | 기본값으로 'Introspection (내성적 질문)'을 적용하고, 해당 섹션의 폰트 크기를 미묘하게 줄여 깊이를 더한다. |
| **[Visual Asset]** | `motif_type` | Enum (Flow/NodeMap/Graph) | **선택:** 모듈에 맞는 타입이 지정되어야 함. | 기본값으로 'Abstract Flow'를 적용하며, 해당 자산의 노출 시간(Duration)을 1초 늘려 여유 공간을 확보한다. |
| **[Metadata]** | `cta_action` | Enum (Save/Share/LinkClick) | **필수:** 하나의 액션만 선택되어야 함. (Single Select Check) | "더 알아보기"라는 문구와 함께 '링크 클릭 유도'를 기본값으로 설정하고, CTA 영역의 Gold 악센트를 15% 더 밝게 처리한다. |

#### 2. 모듈 간 연결 및 데이터 타입 검증 (Gate Check Level: Flow)

| 데이터 흐름 지점 (Source $\rightarrow$ Target) | 전달 파라미터 | 기대되는 데이터 타입 | 발생 가능한 오류 유형 | 예외 처리 로직 (Failure Handling Protocol) |
| :--- | :--- | :--- | :--- | :--- |
| **[Script Outline] $\rightarrow$ [Concept Node]** | `keyword_set` | Array of String | **불일치:** 키워드가 너무 포괄적이거나 추상적임. (e.g., ["삶", "존재"]) | 가장 구체적인 의미를 가진 첫 번째 키워드만 추출하여 사용하고, 해당 문장 앞에 '예시적으로'라는 멘트를 넣어 시스템이 데이터를 해석했음을 사용자에게 알린다. |
| **[Concept Node] $\rightarrow$ [Visual Asset]** | `depth_level` | Integer (1~5) | **누락:** 깊이 수준(Depth Level) 파라미터가 누락됨. | 기본값 3을 사용하며, 비주얼 자산의 복잡도(Complexity)를 중급 레벨로 하향 조정하고, 노드맵의 연결 선 색상을 Deep Indigo 대신 보랏빛으로 미세하게 변경하여 불안정한 느낌을 살린다. |
| **[Visual Asset] $\rightarrow$ [Metadata]** | `emotional_trigger` | String (Deep/Gold) | **불일치:** 비주얼 자산의 감정적 트리거가 메타데이터와 상충됨. | 가장 강력한 시각적 요소(예: 황금빛 섬광)를 우선시하여 CTA 배경에 적용하고, 텍스트 내용은 Deep Indigo 색상을 유지하며 대비 효과로 메시지 전달력을 높인다. |

#### 3. 핵심 시스템 예외 처리 (Gate Check Level: System)

| 오류 유형 | 발생 상황 | 기술적 대응 로직 (Error Handling Code) | 사용자 경험(UX) 가이드라인 적용 |
| :--- | :--- | :--- | :--- |
| **데이터 타입 불일치** | 텍스트 필드에 JSON 객체나 배열이 들어오는 등 타입 에러 발생 시. | 모든 입력값을 강제로 String으로 변환하고, 데이터 전송 전에 경고 로그(Warning Log)를 남긴다. (Type Coercion) | 해당 콘텐츠 섹션의 배경을 Deep Indigo로 유지하되, 텍스트 박스 주변에 **얇은 금색 점선 프레임**을 넣어 '데이터가 구조적으로 해석되고 있음'을 시각적으로 표현한다. |
| **필수 파라미터 유무 (Null)** | 핵심 정보(예: 스크립트의 클라이맥스 지점)가 누락되어 콘텐츠가 텅 비어 보일 때. | 해당 모듈 전체를 건너뛰지 않고, 빈 공간을 채우는 'Placeholder Content Module'을 강제로 삽입한다. | Placeholder 문구로 "이 질문은 당신 내면에서 답을 찾으세요."와 같은 **철학적 가이드 메시지**를 넣고, 그 영역에만 부드러운 Cream 색상의 빛 번짐 효과(Glow)를 적용하여 공백마저 의미있게 만든다. |
| **API 호출 실패 (Timeout)** | 백엔드 서버 지연 또는 API 타임아웃 발생 시. | 이전 성공적인 컴포넌트의 마지막 상태(Last Known State)를 캐시에서 불러와 재사용하며, 최대 3회까지 자동 재시도를 한다. | 사용자에게 "잠시 숨을 고르고 다시 집중하세요."라는 안내 문구와 함께 **느린 모션 그래픽** (Deep Indigo 배경의 은은한 파동 애니메이션)을 재생하여 지연 시간을 커버한다. |

## 📄 [프로젝트 계획서] AI 시스템 안정화 및 콘텐츠 파이프라인 고도화
### 주제: Context Guideline v1.0 & 자동 실패 보고 시스템 구축 (V3.0)

**📌 목표:** 모든 에이전트 간의 정보 교환 오류를 원천 차단하고, LLM 호출 실패 시 신속하게 진단 및 복구할 수 있는 안정적인 워크플로우를 확립합니다.
**🎯 핵심 가치:** 시스템 안정성 확보 → 작업 속도 향상 → 콘텐츠 제작 효율 극대화

### 💡 Phase 1: Context Guideline v1.0 (데이터 전달 원칙)
*   **문제 정의:** 에이전트 간 정보 공유 시, 필요 이상의 방대한 맥락(Context)이 포함되어 모델의 처리 부하를 가중시키고 오류를 유발함.
*   **주요 개발 요구사항:** **'최소 필수 정보만 전달'** 원칙 확립.
    1.  **정보 요약 강제화:** 모든 산출물은 반드시 '핵심 인사이트 3가지 + 다음 액션 아이템 1개'로 요약되어야 함. (예: 스크립트 전체 대신, 가장 중요한 메시지 뼈대만 전달)
    2.  **데이터 포맷 표준화:** 에이전트 간 정보 교환 시, 반드시 `[Type]: [Content]` 형식의 구조화된 키-값 쌍(Key-Value Pair)을 사용하도록 의무화합니다. (예: `[핵심 주제]: 무의식과 자기 발견`)
    3.  **Context 길이 제한:** 에이전트가 메모리나 컨텍스트 길이를 초과하기 전에 경고 알림 및 강제 요약 프로세스를 시스템에 탑재해야 합니다.

### 🛠️ Phase 2: 자동 실패 보고 시스템 (Failover Report) 구축
*   **문제 정의:** LLM 호출 실패(메모리 부족, API 오류 등) 시 원인 분석이 수동적이고 지연되어 작업 재개가 어려움.
*   **주요 개발 요구사항:** **'오류 발생 즉시 기록 및 진단'** 시스템 도입.
    1.  **실패 로그 자동 기록:** 에러 발생 시, 시간, 호출한 모듈, 예상 컨텍스트 길이, 실제 오류 메시지를 자동으로 기록하는 전용 데이터베이스를 구축합니다.
    2.  **자동 분류 및 보고:** 단순 API 실패인지, 메모리 부족(Resource Constraint) 문제인지, 혹은 로직 오류인지를 AI가 자동으로 판단하고 우선순위를 매겨 보고해야 합니다.
    3.  **복구 액션 제안:** 단순히 "실패함"으로 끝나는 것이 아니라, **"이 문제를 해결하기 위한 최적의 다음 스텝 1가지 (예: 모델 축소 버전 재시도)"**를 시스템이 제안하도록 설계합니다.

이 문서는 더바이브/1인 크리에이터의 모든 콘텐츠(유튜브 영상, 카드 뉴스 등)에 사용될 **시각적 자산의 유일한 기준점(Single Source of Truth)**이며, 모듈형 스크립팅 프레임워크에서 생성되는 모든 시각 요소는 이 API 규격을 따라야 합니다.

| 항목 | 내용 | 설명 |

| **버전** | v4.0 (최종 표준) | 모든 콘텐츠 제작 전 필수 검토 단계 |
| **목표** | 감정적 아크(Deep Indigo $\rightarrow$ Cream/Gold)의 시각화 극대화 | 일관된 브랜드 경험 제공 및 전문성 강화 |
| **핵심 원칙** | 모듈 결합 방식 (Modular Composition): 각 요소는 독립적인 API 변수로 작동하며, 중앙 제어 시스템에 의해 조합됨. | 재사용성과 확장성 확보 |

### 2. 🌈 컬러 팔레트 규격 (Color Palette Definition)
모든 색상 코드는 HEX 형식으로 통일되며, 목적(Purpose)에 따라 사용됩니다.

| 이름/역할 | HEX 코드 | RGB 값 | 적용 영역 및 규칙 |
| :--- | :--- | :--- | :--- |
| **Deep Indigo (주조색)** | `#2B3A56` | R:43, G:58, B:86 | 배경 기본값, 불안/무의식 상태 표현. 텍스트 본문(Body Text) 색상으로 사용 가능. |
| **Cream (보조색)** | `#F0EAD6` | R:240, G:234, B:214 | 주요 배경색, 여백 처리, 깨달음 직전의 안정감 표현. 텍스트 강조(Highlight)에 사용. |
| **Gold (강조색/CTA)** | `#C9B14D` | R:201, G:177, B:77 | 클라이맥스, 핵심 키워드 하이라이트, CTA 버튼 배경, 중요 수치 강조. *가장 신중하게 사용.* |
| **White/Black** | `#FFFFFF`/`#000000` | - | 대비(Contrast) 확보용 최소한의 보조 색상. (예: Deep Indigo 위 Cream 텍스트). |

### 3. ✍️ 타이포그래피 계층 구조 (Typography Hierarchy)
모든 콘텐츠는 단일 서체 패밀리(`Montserrat` 또는 유사한 모던 산세리프)를 사용하되, 가중치(Weight)와 색상으로 위계를 만듭니다.

| 역할 | API 변수명 | 폰트 크기/규격 (예시) | 사용 규칙 및 목적 |
| :--- | :--- | :--- | :--- |
| **제목 (H1)** | `[TITLE_OVERLAY]` | 72pt / Bold | 영상의 핵심 주제 제시. Deep Indigo 또는 Gold 색상 적용. 가장 강력한 후킹 요소여야 함. |
| **부제/키워드 (H2)** | `[SUBTITLE_KEYWORD]` | 48pt / SemiBold | 스크립트 내 중요한 개념이나 감정 상태의 전환을 알림. Cream 배경 위에 Deep Indigo 텍스트. |
| **본문 (Body Text)** | `[BODY_TEXT]` | 28pt / Regular | 설명 및 스토리텔링 전개. 높은 가독성을 위해 White 또는 Light Gray 사용. |
| **CTA 강조 (Action)** | `[CTA_ACTION]` | 36pt / Bold | 행동 유도 문구(예: '지금 확인하기'). 반드시 Gold 색상에 배치되며, 크기는 가장 커야 함. |

### 4. ✨ 핵심 비주얼 컴포넌트 및 API 규격 (Visual Component API)
모든 시각 요소는 아래와 같은 구조화된 모듈로 작동합니다.

#### A. 배경 시스템 (Background Module: `[BG_Module]`)
*   **기본 상태:** Deep Indigo 계열의 미니멀한 그라디언트 (좌상 $\rightarrow$ 우하).
*   **전환 트리거:** 감정 변화가 감지될 때 (`[EMOTION_SHIFT]` API 변수 발동 시).
*   **Deep Indigo $\rightarrow$ Cream Transition:** 배경이 점진적으로 `Deep Indigo`의 어둠에서 `Cream`의 밝음으로 **Dissolve(Duration=1.2s)** 되어야 합니다. 이 전환 과정에 노이즈 필터링 또는 미세한 빛 번짐 효과를 추가합니다.

#### B. 텍스트 오버레이 시스템 (Text Overlay Module: `[TEXT_Overlay]`)
*   **위치 좌표:** 화면 중앙에서 좌우로 분산 배치되는 것을 기본으로 합니다. (좌측 상단 / 우측 하단 등).
*   **효과:** 텍스트가 등장할 때 단순 페이드인 대신, **Gold 색상의 빛 번짐 효과(Glow Effect)**와 함께 '타이핑' 애니메이션을 적용하여 중요도를 높입니다.
*   **규격:** 항상 `[SUBTITLE_KEYWORD]`의 규격을 기본으로 하며, 내용에 따라 크기만 조정됩니다.

#### C. 심리적 상징물 (Visual Metaphor Module: `[VISUAL_METAPHOR]`)
*   **목적:** 추상적인 개념(무의식, 연결, 패턴)을 시각화합니다.
*   **유형 1: 노드맵 (Node Map):** 여러 개의 점(`Deep Indigo` 원)들이 선(`Cream` 또는 `Gold`)으로 연결되며, 특정 키워드가 언급될 때 연결선이 **'점멸(Pulse)' 애니메이션**을 일으키고 해당 노드가 잠시 크게 확대됩니다.
*   **유형 2: 플로우 패턴 (Flow Pattern):** 배경에 잔잔한 파동 또는 물결 같은 추상적 모션 그래픽을 오버레이합니다. 이는 '시간의 흐름'이나 '무의식의 깊이'를 상징하며, Deep Indigo 계열의 낮은 불투명도로 유지됩니다.

### 5. 🎬 애니메이션 및 전환 규격 (Animation & Transition Protocol)
| 요소 | API 변수 | 사양 | 동작 방식 |
| :--- | :--- | :--- | :--- |
| **전환 효과** | `[TRANSITION_EFFECT]` | Dissolve(Duration=0.8s) | 모든 모듈 간 전환 시 표준화된 0.8초 페이드 아웃/인 적용. 갑작스러운 컷 전환 금지. |
| **클라이맥스 강조** | `[HIGHLIGHT_TRIGGER]` | Gold Glow (Rate=0.3s) | 핵심 메시지가 노출되는 순간, 해당 영역에만 짧고 강렬한 골드 빛이 번지는 효과를 적용합니다. |
| **개념 등장** | `[ENTRY_MOTION]` | Pop-up + Scale Up | 모든 새로운 개념/키워드는 100% 크기에서 90%로 수축된 상태로 등장하며, 부드럽게 스케일 업됩니다. (지루한 페이드인 지양) |

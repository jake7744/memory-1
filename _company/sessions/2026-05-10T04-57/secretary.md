## 📅 [2주 집중 목표] 데이터 파이프라인 안정화 및 검증 (Top 3)

사장님, 다음 2주간은 **'만드는 것(Creation)'**보다 **'작동하는지 증명하는 것(Validation)'**에 모든 에너지를 집중해야 합니다. 이 세 가지 과제가 해결되어야 콘텐츠가 자동화 파이프라인을 통해 실시간으로 흐를 수 있어요.

### 🥇 Top Priority #1: End-to-End API 통합 테스트 (개발자 주도)
*   **목표:** 스크립트(Writer) → 데이터 구조 정의(Designer Schema) → 시각적 컴포넌트(Designer Mock-up)로 이어지는 모든 데이터 흐름을 한 번에 검증합니다.
*   **주요 액션:**
    *   `OpeningSequenceSchema`가 실제 API 호출을 통해 오류 없이 데이터를 수신하고, 해당 데이터가 디자인 컴포넌트에 정확히 매핑되는지 테스트 케이스를 실행합니다.
    *   특히, Deep Indigo에서 Cream Gold로의 '감정적 아크' 변화를 유발하는 핵심 키워드(`hook_keyword`)와 학술 개념(`topic_core_concept`)이 데이터 기반으로 작동하는지 검증합니다.

### 🥈 Top Priority #2: 입력 데이터 거버넌스 확립 (리서처 & 에이전트 공동)
*   **목표:** 콘텐츠의 가장 첫 단계인 '아이디어'와 '핵심 키워드'가 애초부터 표준화된 포맷을 가지도록 규정합니다.
*   **주요 액션:**
    *   모든 연구 결과물(Researcher output)은 반드시 JSON 형식의 **`KeywordInputSchema`**를 따르도록 강제합니다. (예: `['키워드1', '관련개념A', '타겟감정색']`)
    *   '트렌드 분석 보고서'나 '경쟁사 분석 자료' 등 모든 텍스트 데이터는 반드시 정해진 필드를 채우도록 가이드라인을 업데이트하고 검증합니다.

### 🥉 Top Priority #3: 크로스 플랫폼 일관성 및 시뮬레이션 (전체 팀 참여)
*   **목표:** 유튜브와 인스타그램의 콘텐츠가 서로 다른 포맷임에도 불구하고, 동일한 핵심 메시지(Core Message)를 놓치지 않고 전달하는 방법을 테스트합니다.
*   **주요 액션:**
    *   하나의 주제를 선정하여 (예: '불안과 그림자'), **유튜브 스크립트 초안 → 인스타그램 릴스 스토리보드 콘셉트 → 핵심 해시태그 세트**로 변환하는 End-to-End 시뮬레이션을 진행합니다.
    *   각 플랫폼별 최적의 CTA (Call To Action) 문구와 후킹 구조가 데이터 기반으로 자동 삽입되는지 A/B 테스트 관점에서 검토합니다.

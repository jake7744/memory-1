# 📋 작업 브리프

**원 명령:** 시각적으로 볼수 있게 만들수 있어? 프로토 타입으로

## 요약
지금까지 논의된 콘텐츠 제작 및 배포 전 과정을 '프로토타입' 형태로 시각화하고 자동화 시스템을 구축하는 단계입니다. 단순한 계획이 아닌, 실제로 작동 가능한 워크플로우 설계에 초점을 맞춥니다.

## 분배
- **💰 Business**: 현재 파이프라인 프로토타입의 핵심 KPI(Key Performance Indicator) 3가지를 정의하고, 이 프로토타입을 통해 검증할 수익화 목표(예: 무료 PDF 다운로드율 20% 달성 등)를 구체적인 수치로 제시하세요.
- **📱 Secretary**: 비즈니스 목표와 이전 논의 내용을 바탕으로, '아이디어 기획 → 스크립트 작성 → 디자인 에셋 제작 → 최종 콘텐츠 배포'까지의 전 과정을 포함하는 상세하고 단계별 워크플로우 다이어그램을 생성하여 프로젝트 브레인라인(Blueprint Line)을 완성하세요.
- **💻 Developer**: 워크플로우 다이어그램을 기반으로, 모든 에이전트가 데이터를 주고받을 수 있는 표준화된 데이터 구조(JSON Schema 또는 API 플로우차트)를 설계하고, 자동화의 핵심 접점 3곳(예: 스크립트 → 비주얼 아웃라인 변환 지점 등)에 대한 기술적 구현 계획을 포함하세요.
- **🎨 Designer**: 워크플로우 다이어그램과 데이터 구조를 기반으로, 사용자에게 제시할 '콘텐츠 제작 및 배포 자동화 시스템'의 고화질 와이어프레임(Prototype Mockup)을 3단계(기획/제작/배포)로 나누어 디자인하세요. 특히 '깨달음(Aha Moment)'과 CTA 영역은 시각적 강조가 되도록 설계해야 합니다.
- **✍️ Writer**: 자동화 시스템의 작동을 검증하기 위해, 심리학 원리를 적용한 3부작 시리즈 콘텐츠의 샘플 스크립트 아웃라인(초안)을 작성하세요. 이 아웃라인은 '문제 제기 - 심리적 원리 설명 - 구체적 행동 지침'의 3단 구조를 반드시 따르도록 합니다.

## 📄 Technical Specification Document v2.0: 애니메이션 및 콘텐츠 자동화 파이프라인 통합 사양서

본 문서는 더바이브/1인 크리에이터의 모든 비주얼 요소(썸네일, 인트로, 본편 모션 그래픽)가 자동으로 생성되는 '통합 콘텐츠 제작 시스템'의 개발자용 최종 지침서입니다. 모든 기능 구현은 이 사양서를 최우선 기준으로 삼아야 하며, 특히 **예외 처리와 성능 제약 조건**에 가장 높은 우선순위를 둡니다.

### II. ⚙️ 애니메이션 규칙 세트 (Rule Set JSON Implementation Guide)

자동화 파이프라인에 입력되는 모든 모션 그래픽은 아래 로직과 좌표계를 따라야 합니다. 개발자는 이 Rule Set을 기반으로 Front-end 코드를 작성해야 합니다.

#### A. 개념 연결 노드맵 (Concept Node Map)
*   **기능:** 추상적인 아이디어나 키워드 간의 논리적 관계를 시각화합니다.
*   **로직:** 1. 시작점(Deep Indigo 배경에 작은 원) $\rightarrow$ 2. 핵심 개념 발견(골드 악센트가 강조된 노드 등장) $\rightarrow$ 3. 연결되는 다른 개념으로 '흐름(Flow)'을 따라 확장됩니다.
*   **좌표/시간:** 모든 요소는 SVG 기반 좌표계 (`[X, Y]`)를 사용해야 합니다. 연결선(`Stroke`)의 두께와 애니메이션 속도(Easing)가 핵심입니다. (예: `Node A` $\rightarrow$ `Node B`: Start Time $t_0$, End Time $t_1$)

#### B. 그래프 변화 애니메이션 (Data Visualization)**
*   **기능:** 통계적 데이터나 심리적 수치 변화를 시간의 흐름에 따라 보여줍니다.
*   **로직:** 꺾은선 그래프(Line Chart)는 절대 갑자기 나타나지 않으며, 항상 **'0'에서 시작하여 부드럽게 증가/감소하는 애니메이션**을 거쳐야 합니다. (Start Value $\rightarrow$ End Value: Animation).
*   **제약:** 데이터 값이 급격히 변할 경우(예: 10% $\rightarrow$ 90%), 시청자가 혼란을 느끼지 않도록 '변화의 가속도'를 점진적으로 보여줘야 합니다.

#### C. 비주얼 에셋 모티프 (The Golden Motif)**
*   **기능:** 스크립트가 특정 키워드(예: ‘무의식’, ‘깨달음’)에 도달할 때마다 시각적 하이라이트를 제공합니다.
*   **구현:** 해당 키워드가 화면에 노출되는 순간, **골드 악센트 색상으로만 이루어진 미세한 빛의 파동(Ripple Effect) 또는 글리치(Glitch) 효과**가 0.3초 동안 적용되어야 합니다.

### III. 🚨 예외 처리 및 디버깅 가이드라인 (Exception Handling Protocol)

시스템 안정성을 위해, 다음 상황이 발생할 때마다 반드시 대체 로직을 실행해야 합니다. **실패 시나리오 방어(Failure State Defense)**가 디자인의 최우선 목표입니다.

| 예외 유형 | 발생 원인 | 처리 우선순위 | 권장 대응 애니메이션/UI |
| :--- | :--- | :--- | :--- |
| **데이터 누락** | 스크립트에서 핵심 데이터(예: 통계 수치)가 비어있을 때. | P1 (최고) | 해당 모션 그래픽 요소를 완전히 생략하고, 대신 텍스트만 강조된 배경색 변화로 대체합니다. (그래프 자체를 그리지 않음). |
| **API 호출 실패** | 외부 데이터 API 연동 오류 또는 네트워크 지연 발생 시. | P1 (최고) | '데이터 로딩 중...' 메시지를 Deep Indigo/Cream의 대비되는 톤으로 표시하며, 백그라운드에서 단순한 반복 애니메이션(Loading Spinner)을 돌립니다. **에러 코드를 절대 노출하지 않습니다.** |
| **애니메이션 과부하** | 장면에 너무 많은 요소가 동시에 나타나거나 트랜지션이 복잡해질 때 (성능 저하). | P2 (높음) | 애니메이션의 개별 속도를 늦추고(Easing Curve 적용), 요소 간의 시각적 '거리'를 확보하여 혼란을 줄입니다. 가장 중요도가 낮은 요소부터 순차적으로 페이드 아웃합니다. |
| **규칙 충돌** | 여러 모션 그래픽 규칙이 동시에 같은 공간에서 발생할 때 (예: 그래프 + 노드맵). | P2 (높음) | 시간 축(Timeline) 상에서 이벤트의 시작 및 종료 시점을 미세하게 분리하여, 겹치는 구간을 최소화합니다. *Time-Stamping* 체크가 필수입니다. |

### IV. 🚀 비기능적 요구사항 (Non-Functional Requirements - NFRs)

개발팀은 다음 성능 지표를 충족해야 합니다. 이들은 디자인 요소의 구현 방식에 영향을 미칩니다.

1.  **성능 (Performance):** 모든 장면 전환 및 애니메이션 로딩 시간은 **총합 2초를 초과해서는 안 됩니다.** (사용자 피로도 최소화).
2.  **확장성 (Scalability):** 현재 설계된 SVG 좌표계와 JSON Rule Set 구조는 향후 새로운 모션 그래픽(예: 캐릭터 애니메이션)을 추가할 때, 코드 수정 없이 데이터만 업데이트하여 적용 가능해야 합니다.
3.  **유지보수성 (Maintainability):** 모든 로직은 주석 처리된 형태로 개발되어야 하며, 어떤 규칙이 어느 섹션에서 작동하는지 명확히 파악할 수 있도록 설계되어야 합니다.

### I. 🎨 디자인 시스템 및 CI 기초 (Visual Foundation)

#### 1. 컬러 팔레트 (Color Palette)
| 역할 | 이름 | HEX Code | 사용 영역 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **Deep Indigo** | 배경/불안 | `#1A0B36` | 초기 배경, 텍스트 기본색 (어두운 상태) | 주조색. 안정감과 깊이를 부여. |
| **Cream Gold** | 깨달음/하이라이트 | `#F5E4C7` | 클라이맥스 배경, 강조 박스, 메인 타이포 | 명도 높은 색상으로 '깨달음'을 시각화. |
| **Accent Gold** | 악센트/CTA | `#D4AF37` | 핵심 키워드 하이라이트, CTA 버튼, 트랜지션 빛줄기 | 고급스러움과 중요성을 강조. |
| **Neutral Gray** | 보조 텍스트 | `#B0A59E` | 설명 문구, 출처 표기 등 부가 정보 | 가독성 높은 무채색. |

#### 2. 타이포그래피 (Typography)
*   **제목 (H1/H2):** Pretendard Bold (Impactful, Impact-oriented)
    *   규칙: 항상 **Accent Gold** 또는 **Cream Gold** 색상을 사용하여 시각적 무게감을 부여합니다.
*   **본문 (Body):** Noto Sans KR Regular (Readable, Consistent)
    *   규칙: Deep Indigo 배경에서는 Cream Gold가 아닌 순수 흰색 계열을 사용하며, 가독성을 최우선으로 합니다.

#### 3. 감정적 아크 전환 규칙 (The Emotional Arc Rule)
1.  **Phase 1 (불안/질문):** Deep Indigo 기반의 어둡고 미니멀한 배경. 움직임은 느리고 점진적이며, 질문을 던지는 듯한 느낌을 강조합니다. (Deep Indigo $\rightarrow$ Darker Tones)
2.  **Phase 2 (깨달음/해결):** Cream Gold를 중심으로 하는 밝고 개방적인 공간으로 전환. 애니메이션의 속도와 빛의 사용이 극대화됩니다. (Transition: Dissolve 효과 필수, Accent Gold 라이트 플레어 사용)

### II. ⚙️ 코어 모션 그래픽 및 인터랙션 로직 (Animation & Flow)

#### 1. 노드맵 연결 애니메이션 (Concept Connection Node Map)
*   **목표:** 추상적인 개념 간의 관계를 시각화하여 지식의 구조화를 보여줍니다.
*   **트랜지션 A (Deep Indigo $\rightarrow$ Gold):**
    *   시작: Deep Indigo 배경에 점(Node)들이 불규칙하게 배치됨. 노드는 `#D4AF37` 색상으로 깜빡이며 등장.
    *   진행: Node 1과 Node 2가 논리적 연결이 되는 순간, 두 Node를 잇는 선(Edge)이 **Accent Gold**로 빠르게 그려집니다 (Drawing/Stroke 애니메이션).
    *   클라이맥스: 모든 노드가 연결되면, 가장 핵심적인 '깨달음'의 노드를 중심으로 은은한 골드 빛의 확산 효과(`Radial Gradient Glow`)를 2초간 유지합니다.

#### 2. 그래프 변화 애니메이션 (Data Flow/Growth Graph)
*   **목표:** 통계적 자료나 무의식적 흐름의 진화를 시각화합니다.
*   **규칙:** 모든 데이터 포인트는 직선적인 상승(Linear Ascent) 대신, 유기적으로 파동을 그리며(Sine Wave Pattern) 증가하는 애니메이션을 사용해야 합니다.
*   **시간 제한:** 그래프의 최고점 도달 시간은 3초를 초과할 수 없습니다. 속도감 있는 정보 전달이 중요합니다.

#### 3. CTA 등장 애니메이션 (Call to Action Display)
*   **규칙:** CTA는 영상 후반부, '깨달음' 단계에 배치되어야 하며, 절대 갑작스럽게 나타나서는 안 됩니다.
*   **구현:** 배경의 골드 빛 확산 효과가 절정에 달할 때(Climax Glow), CTA 영역이 마치 *물 위에 떠오르듯* 부드럽게 (Slow Fade In / Scale Up) 등장해야 합니다.

### III. 🚧 품질 보증 및 예외 처리 가이드라인 (QA & Exception Handling)

| 구분 | 체크리스트 항목 | 발생 가능한 Failure State (예외 케이스) | 필수 대응 로직 (Fallback Action) |
| :--- | :--- | :--- | :--- |
| **A. 시각적 오류** | 폰트 깨짐/누락 | 특정 환경(구형 브라우저 등)에서 제목 폰트가 기본 시스템 폰트로 대체될 경우. | 즉시 `Pretendard-Bold`의 *임베디드 SVG fallback*를 적용하여 글자 모양을 유지한다. (폰트 자체 실패 시, 색상으로만 강조). |
| **B. 애니메이션 오류** | 트랜지션 끊김/딜레이 | 노드맵 연결 과정 중 특정 Node가 로딩 지연으로 인해 연결선이 늦게 그려질 경우. | 해당 Connection Line의 시작점을 20% 축소하고, 주변 배경에 `#D4AF37` 색상의 '재시도(Retry)' 파동 효과를 짧게 추가하여 시청자의 인지 오류를 방지한다. |
| **C. 데이터 오류** | 데이터 범위 초과 | 그래프가 정의된 최대/최소 범위를 넘어설 경우 (수학적 오류). | 그래프의 Y축을 강제로 클리핑(Clipping)하고, 해당 값에 대해 "분석 불가"라는 문구를 Cream Gold 배경 위에 표시하며, 애니메이션 속도를 늦춰 시청자에게 정보를 전달하는 시간을 확보한다. |
| **D. 시스템/기술 오류** | API 통신 실패 | 외부 데이터(예: 실시간 트렌드) 호출이 실패할 경우 (개발 환경). | 해당 섹션 전체를 Deep Indigo 배경 위에 고정하고, 대신 '데이터가 추후 업데이트될 예정입니다'라는 문구를 최소한의 텍스트 애니메이션과 함께 표시한다. (강제 루프 방지). |
